# Release Notes - v3.2.1

## Breaking Changes
- **Profile save target changed**: `--save backend` has been replaced by `-p backend --save`.
- **Profile save semantics clarified**: `--save` now saves explicit options back to the active profile, merging by default.

## New Features
- **Full profile management**:
  - Added `profile list` with `profile ls` as an alias.
  - Added `profile show <name>` with `--json` support.
  - Added `profile create <name> [OPTIONS]`.
  - Added `profile update <name> [OPTIONS]` with `--replace`.
  - Added `profile delete <name>` with `profile rm` as an alias.
  - Added `profile rename <old> <new>`.
- **Replace mode**: Added `--replace` for rebuilding the active profile from the current explicit options.

## Improvements
- **Safer configuration handling**: Invalid `.onesourcerc` files are no longer silently overwritten.
- **Profile validation**: Profile names are now restricted to letters, numbers, `.`, `_`, and `-`.
- **Stable profile output**: Profile listing is sorted by name.
- **Profile display**: `profile show` now provides a static readable view grouped by content, tree, and behavior settings.
- **CI/CD split**:
  - `Rust Test` now handles formatting, clippy, unit tests, and debug builds.
  - `GitHub CLI Integration Test` now handles CLI workflow scenarios.
  - Release publishing now runs a preflight gate before uploading assets.

## Fixes
- Fixed a Clippy failure in profile sorting by using `sort_by_key`.

## Notes
- The `.onesourcerc` JSON format remains compatible.
- `profile desc` is kept as a hidden legacy command; use `profile update <name> --desc "..."` going forward.
- This release contains the profile UX stabilization work intended for v3.2.0, plus the CI fix needed for release preflight.
