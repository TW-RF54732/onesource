# Release Notes - v3.5.0

## Highlights

- Packaging is now deterministic: selected files are sorted, the active output file is always excluded, and repeated runs no longer package a previous `.onesource` file into itself.
- Output is generated through a temporary sibling file and replaced only after a successful render, preserving the previous output when generation fails.
- Added clearer scan diagnostics: invalid glob patterns return an error, non-UTF-8 text emits a warning, and skipped/read failures are summarized.
- `Estimated Output Tokens` now includes the directory tree and file framing in addition to content.
- `explain` now presents each requested path in a distinct `Path N/Total` section and reports more cases, including directories, root-external paths, lossy UTF-8, and the active output file.

## Security and behavior changes

- Expanded the default blacklist to cover `.env.*`, common private-key and credential files, previous `.onesource` outputs, and internal temporary/backup files. Use `--no-blacklist` only after reviewing the resulting context.
- Symbolic links whose targets are outside the scan root are no longer read into file content.
- The max-size threshold is now exact: files larger than `--max-size × 1024` bytes are skipped.
- `--no-ignore` is inherited by the directory tree unless `--tree-no-ignore` is explicitly set. Use `--tree-no-ignore=false` to keep tree ignore filtering enabled while content ignores are disabled.
- Passing a file as the scan root now returns an error instead of generating an empty file path entry.

## Compatibility notes

- Existing CLI option names and `.onesourcerc` schema are unchanged.
- Scripts parsing the old `Total Tokens:` line must use `Estimated Output Tokens:` instead.
- Paths containing XML-special characters are now escaped in `<file path="…">` attributes.
