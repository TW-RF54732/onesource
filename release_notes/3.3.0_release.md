# Release Notes - v3.3.0

## New Features
- **Self-update command**: Added `onesource update` so users can update the currently running binary without finding the original install script again.
- **Portable update support**: Self-update targets the executable path reported by the running process, so portable binaries and custom install locations can be updated in place.

## Improvements
- **Installer-independent updates**: The README now documents `onesource update` as the normal update path after the first install.
- **Cross-platform release asset selection**: The updater selects the matching GitHub release asset for Windows, Linux, and macOS.

## Platform Notes
- On Windows, the updater downloads the new executable and starts a background PowerShell replacement step after the current process exits, because Windows cannot overwrite a running `.exe`.
- On Linux and macOS, the updater replaces the executable directly and preserves executable permissions on the downloaded asset.

## CI/CD
- This release is intended to be published by pushing the `v3.3.0` tag. The existing GitHub Actions release workflow will run preflight checks, build release binaries for Windows/Linux/macOS, and attach assets using this release note file as the GitHub Release body.
