# Onesource Token Calculation Update[cite: 2]

## Modifications[cite: 2]

### Modified Files[cite: 2]
- `src/main.rs`[cite: 2]

### Added Dependencies[cite: 2]
Utilized the existing `tiktoken-rs = "0.11.0"` dependency to calculate tokens.[cite: 2]

## Code Changes[cite: 2]

### 1. Added `use` Statement[cite: 2]
```rust
use tiktoken_rs::cl100k_base;
```

### 2. Added Statistics Struct[cite: 2]
```rust
#[derive(Debug, Default)]
struct FileStats {
    file_count: u32,
    total_tokens: usize,
}
```

### 3. Modified `rw_file` Function[cite: 2]
- **Before**: `fn rw_file<W: Write>(args:&AppConfig, writer:&mut W)`[cite: 2]
- **After**: `fn rw_file<W: Write>(args:&AppConfig, writer:&mut W) -> FileStats`[cite: 2]

#### Key Changes:[cite: 2]
1. Initialize tokenizer:[cite: 2]
   ```rust
   let bpe = cl100k_base().expect("Failed to load tokenizer");
   ```
2. Track statistics:[cite: 2]
   ```rust
   let mut stats = FileStats::default();
   ```
3. Calculate tokens for each file:[cite: 2]
   ```rust
   let tokens = bpe.encode_with_special_tokens(&content);
   let token_count = tokens.len();
   stats.total_tokens += token_count;
   ```
4. Display token count per file:[cite: 2]
   ```rust
   println!("  + {} ({} tokens)", rel_path.display(), token_count);
   ```
5. Display totals:[cite: 2]
   ```rust
   println!("Files Processed: {}", stats.file_count);
   println!("Total Tokens: {}", stats.total_tokens);
   ```
6. Return statistics data:[cite: 2]
   
```rust
   stats
   ```

### 4. Token Calculation in Dry Run Mode[cite: 2]
The dry run mode will also read files and calculate tokens:[cite: 2]
```rust
if args.dry_run {
    if let Ok(bytes) = std::fs::read(entry.path()) {
        let content = String::from_utf8_lossy(&bytes);
        let tokens = bpe.encode_with_special_tokens(&content);
        let token_count = tokens.len();
        stats.total_tokens += token_count;
        println!("[EXPECT] {} ({} tokens)", rel_path.display(), token_count);
    }
    // ...
}
```

## Technical Details[cite: 2]

### Tokenizer[cite: 2]
- Utilizes the `cl100k_base` encoder.[cite: 2]
- This is the tokenizer used by OpenAI's GPT-4 and GPT-3.5-turbo.[cite: 2]
- Includes calculation for special tokens.[cite: 2]

### Performance Impact[cite: 2]
- Calculation is performed on-the-fly right after file reading, requiring no additional reads.[cite: 2]
- Minimal impact on overall execution time.[cite: 2]
- Memory usage increase is negligible.[cite: 2]

### Compatibility[cite: 2]
- Fully backward compatible.[cite: 2]
- All existing arguments and features operate normally.[cite: 2]
- The output format remains unchanged, aside from the addition of token information.[cite: 2]

## Why is this useful?[cite: 2]

1. **Estimate API Costs**: Knowing the total token count helps estimate the expense of using Claude/GPT APIs.[cite: 2]
2. **Check Context Limits**: Ensures the content does not exceed the model's context window (e.g., Claude 3.5 Sonnet has a 200K token limit).[cite: 2]
3. **Optimize Content**: If the token count is too high, `include`/`exclude` parameters can be adjusted to reduce context size.[cite: 2]
