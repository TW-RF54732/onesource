# Release Notes - v3.0.0

## Breaking Changes
The configuration format for `.onesourcerc` has been completely restructured to support multiple profiles. Legacy flat JSON configurations are no longer supported. Upon detection of an old format, the tool will now exit with a migration warning.

## New Features
- Profile System: Support for multiple named configuration profiles within a single `.onesourcerc` file.
- Named Saving: The `--save` flag now accepts an optional profile name (e.g., `--save backend`). Defaults to "default".
- Profile Loading: Use `-p` or `--profile` to load a specific configuration block.
- Profile Management CLI: Added `onesource profile ls` to list available profiles and their descriptions.
- Machine-Readable Output: Added `onesource profile ls --json` for integration with external scripts and AI agents.
- Description Field: Each profile now supports an optional description field for better context management.

## Improvements
- Parameter Merging: Refined priority logic ensures CLI arguments always override profile settings, which in turn override internal defaults.
- Automated Testing: Introduced a comprehensive CI/CD suite covering profile management, parameter priority, and backward compatibility checks.
- Version Bump: Formally moved to v3.0.0 to reflect significant architectural changes.

## Migration Guide
To migrate from v2.x, either delete your `.onesourcerc` and regenerate it using `--save`, or wrap your existing configuration in a "profiles" and "default" key:

Old format:
{
  "include": "*.rs"
}

New format:
{
  "profiles": {
    "default": {
      "include": "*.rs"
    }
  }
}
