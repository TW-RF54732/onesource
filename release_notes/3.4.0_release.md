# Release Notes - v3.4.0

## Features
- Added `onesource explain <PATHS...>` as a debug command for checking why specific files or folders are included, excluded, or blocked.
- `explain` reports content and tree decisions separately, so profile and tree-only filters are easier to understand.
- `explain` supports the same scan-related flags and profile loading behavior as normal runs, including `--profile`, `--no-config`, `--include`, `--exclude`, `--no-ignore`, `--no-blacklist`, tree filters, `--no-tree`, and `--max-size`.

## Notes
- `explain` only reports paths explicitly requested by the user. It does not add skipped path details to `--dry-run`, which avoids accidentally listing sensitive ignored paths.
- Ignore-filter explanations follow the normal walker behavior for parity with real packaging.

## Testing
- Ran `cargo fmt`.
- Ran `cargo test`.
- Ran `cargo build`.
- Smoke tested `onesource explain` with include/tree overrides, `--no-blacklist`, `--no-config`, and `--no-tree`.
